💡 권한 부여 방식
1️⃣ 회원가입 시 플랜 선택 → MariaDB users.plan 컬럼에 저장
2️⃣ 로그인 → JWT 토큰 발급 (토큰 안에 plan 정보 포함)
3️⃣ API 요청마다 토큰 검증 → 플랜에 맞는 기능만 허용
4️⃣ 관리자가 DB에서 직접 플랜 변경 가능 → 즉시 적용

🔄 로그인 → 권한 확인 전체 흐름

① 회원가입
POST /api/register
email + password + plan 전송
↓
② 암호화
SHA-256 해시 처리
원본 비밀번호는 절대 DB에 저장 안 함!
↓
③ DB 저장
MariaDB users 테이블 INSERT
id / email / password(해시) / plan / created_at
↓
④ 로그인
POST /api/login → JWT 토큰 발급
토큰 안에 user_id + plan 정보 포함
↓
⑤ API 요청
Header: Authorization: Bearer {토큰}
모든 요청에 토큰 첨부
↓
⑥ 권한 확인
check_permission(plan, feature)
플랜에 맞는 기능인지 서버에서 확인
↓
✅ 허용 → 기능 실행
❌ 거부 → 403 Forbidden

📡 실제 API 사용 방법 (VM 터미널에서 바로 테스트)

① 회원가입

# VM 터미널에서 실행 curl -X POST http://localhost:8000/api/register \ -H "Content-Type: application/json" \ -d '{"email": "jeaje@test.com", "password": "my1234", "plan": "pro"}' # 응답 예시 {"token": "abc123...", "plan": "pro", "user_id": "1", "expire": 1234567890}

② 로그인

curl -X POST http://localhost:8000/api/login \ -H "Content-Type: application/json" \ -d '{"email": "jeaje@test.com", "password": "my1234"}'

③ 토큰으로 AI 분석 실행

curl -X POST http://localhost:8000/api/run \ -H "Authorization: Bearer abc123..." \ -H "Content-Type: application/json" \ -d '{"learning_type": "reinforcement", "user_id": "jeaje"}' # free 플랜이면 → {"error": "권한 없음: free 플랜은 강화학습 사용 불가", "code": 403} # enterprise면 → {"ML결과": {...}, "정확도": 0.87, ...}

④ 관리자 플랜 변경 (MariaDB 직접)

# MariaDB 접속 후 직접 변경 → 즉시 적용! mysql -u vm_user -p vm_project UPDATE users SET plan='enterprise' WHERE email='jeaje@test.com'; SELECT email, plan FROM users; # 확인

⑤ 권한 목록 확인 API

curl http://localhost:8000/api/permissions?plan=pro # 응답 {"plan": "pro", "permissions": ["기본분석","CSV업로드","AI분석","SHAP","PDF다운로드","지도학습","비지도학습"]}
